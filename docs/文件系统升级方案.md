## 1. 项目概述与现状分析

### 1.1 升级目标

新建独立的 **`file_service`** Django App，为用户提供完整的"云盘"文件管理能力，同时作为 AI Agent 的文件知识底座。核心目标：

| # | 目标 | 说明 |
|---|------|------|
| G1 | **独立文件服务** | 从现有 `agent_service` 附件系统中剥离，创建独立的文件存储、解析、检索组件 |
| G2 | **云盘机制** | 为每个用户提供带配额（默认 255MB）的个人云盘，支持文件夹、上传、下载 |
| G3 | **云盘管理页** | `/files` 路由，浏览器界面完成文件 CRUD、批量上传、URL 上传 |
| G4 | **文档预解析** | 文档类文件（PDF/Word/Excel）上传时即解析为 Markdown 并持久化存储，后续调用零重复开销 |
| G5 | **双格式下载** | 用户可下载原始文件或解析后的 Markdown |
| G6 | **MD 在线编辑** | 用户可在管理页编辑已解析的 Markdown，编辑不影响原始文件关联 |
| G7 | **聊天页融合** | 聊天拖拽上传自动同步到云盘默认路径；新增"从云盘加载"选项 |
| G8 | **基础检索 + RAG 预留** | 基于文本匹配的文件内容搜索；预留外部 RAG（阿里云知识库）接口 |
| G9 | **付费扩容预留** | 文件大小上限、存储配额均可配置，数据模型已包含付费等级字段 |

### 1.2 现有附件系统分析

当前附件能力分布在 `agent_service` 中，结构如下：

```
agent_service/
├── attachment_handler.py    # 上传处理 + 多模态格式化
├── model_capabilities.py    # 模型 vision 能力查询
├── models.py                # SessionAttachment 模型
├── parsers/
│   ├── base.py              # BaseParser 抽象基类
│   ├── image_parser.py      # 图片: Base64 + OCR
│   ├── document_parser.py   # 文档: 百度云 → pdfplumber/docx/openpyxl
│   ├── audio_parser.py      # 音频: 百度VOP → faster-whisper
│   └── internal_parser.py   # 内部元素: event/todo/reminder/workflow
└── views_api.py             # upload_attachment / format_attachment_content 等
```

**现有问题**：

1. **会话耦合**：`SessionAttachment` 绑定 `session_id`，文件生命周期跟随会话，无法跨会话复用
2. **重复解析**：同一文件在不同会话上传会重复执行解析流程（PDF 解析 / 百度云调用）
3. **无文件管理**：用户无法浏览、组织、检索已上传的文件，只能在聊天中使用
4. **无持久存储语义**：`media/attachments/` 目录是扁平的按日期分类，无用户隔离、无文件夹概念
5. **无配额管理**：文件大小硬编码 20MB，无用户存储总量限制

### 1.3 新旧系统关系

```
┌─────────────────────────────────────────────────┐
│  file_service (新)                               │
│  ┌──────────┐  ┌──────────┐  ┌───────────────┐ │
│  │UserFolder │  │ UserFile │  │UserStorageQuota│ │
│  └──────────┘  └────┬─────┘  └───────────────┘ │
│                     │                            │
│  永久存储层          │ parsed_markdown            │
│  user_files/<uid>/  │ search_text                │
└─────────────────────┼───────────────────────────┘
                      │
         cloud_file   │  FK (可选)
                      │
┌─────────────────────┼───────────────────────────┐
│  agent_service (现有)│                            │
│               ┌─────┴──────────┐                 │
│               │SessionAttachment│ ← 会话级临时引用│
│               └────────────────┘                 │
│                                                   │
│  parsers/ ← file_service 复用，不重复实现         │
└───────────────────────────────────────────────────┘
```

**关键原则**：

- `UserFile` 是文件的**唯一真相来源**（single source of truth），永久保存
- `SessionAttachment` 降级为对话中的**引用层**，新增 `cloud_file` FK 指向 `UserFile`
- 删除 `SessionAttachment`（回滚）不影响 `UserFile`
- `agent_service/parsers/` 保持原位，`file_service` 通过 import 复用解析器，不复制代码
- 旧的 `media/attachments/` 目录整体废弃（测试环境数据不保留）

### 2.2 UserStorageQuota — 用户配额模型

管理每个用户的存储限制，支持后期付费扩容。

```python
class UserStorageQuota(models.Model):
    """
    用户存储配额

    设计要点：
    - 每个用户一条记录（OneToOne）
    - max_storage_bytes 和 max_file_size 可按用户单独调整（付费升级）
    - used_bytes 由文件上传/删除时主动维护，避免每次实时扫描
    """
    # 默认值（可在 settings.py 覆盖）
    DEFAULT_MAX_STORAGE = 255 * 1024 * 1024      # 255 MB
    DEFAULT_MAX_FILE_SIZE = 20 * 1024 * 1024      # 20 MB

    user = models.OneToOneField(User, on_delete=models.CASCADE, related_name='storage_quota')

    # 配额配置
    max_storage_bytes = models.BigIntegerField(
        default=DEFAULT_MAX_STORAGE,
        help_text="存储空间上限（字节），默认 255MB"
    )
    max_file_size = models.BigIntegerField(
        default=DEFAULT_MAX_FILE_SIZE,
        help_text="单文件大小上限（字节），默认 20MB"
    )

    # 实时用量（由上传/删除操作维护）
    used_bytes = models.BigIntegerField(default=0, help_text="已用存储空间（字节）")
    file_count = models.IntegerField(default=0, help_text="文件总数")

    # 付费等级预留
    tier = models.CharField(
        max_length=20,
        default='free',
        choices=[
            ('free', '免费版'),
            ('basic', '基础版'),
            ('pro', '专业版'),
            ('enterprise', '企业版'),
        ],
        help_text="用户存储等级（预留付费扩容）"
    )
    tier_expires_at = models.DateTimeField(null=True, blank=True, help_text="付费等级到期时间")

    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)

    class Meta:
        verbose_name = "用户存储配额"
        verbose_name_plural = "用户存储配额"

    def __str__(self):
        used_mb = self.used_bytes / (1024 * 1024)
        max_mb = self.max_storage_bytes / (1024 * 1024)
        return f"{self.user.username}: {used_mb:.1f}/{max_mb:.1f} MB ({self.tier})"

    @property
    def remaining_bytes(self):
        return max(0, self.max_storage_bytes - self.used_bytes)

    @property
    def usage_percent(self):
        if self.max_storage_bytes == 0:
            return 100.0
        return round(self.used_bytes / self.max_storage_bytes * 100, 1)

    def can_upload(self, file_size: int) -> tuple[bool, str]:
        """检查是否允许上传指定大小的文件"""
        if file_size > self.max_file_size:
            max_mb = self.max_file_size / (1024 * 1024)
            return False, f"文件大小超过限制（上限 {max_mb:.0f}MB）"
        if self.used_bytes + file_size > self.max_storage_bytes:
            remaining_mb = self.remaining_bytes / (1024 * 1024)
            return False, f"存储空间不足（剩余 {remaining_mb:.1f}MB）"
        return True, ""

    def consume(self, file_size: int):
        """上传文件后增加用量"""
        self.used_bytes += file_size
        self.file_count += 1
        self.save(update_fields=['used_bytes', 'file_count', 'updated_at'])

    def release(self, file_size: int):
        """删除文件后释放用量"""
        self.used_bytes = max(0, self.used_bytes - file_size)
        self.file_count = max(0, self.file_count - 1)
        self.save(update_fields=['used_bytes', 'file_count', 'updated_at'])

    @classmethod
    def get_or_create_for_user(cls, user):
        """获取或创建用户配额记录"""
        quota, _ = cls.objects.get_or_create(user=user)
        return quota
```

### 2.3 UserFolder — 文件夹模型

```python
class UserFolder(models.Model):
    """
    用户文件夹（树形结构）

    使用物化路径（materialized path）方便查询子树。
    示例：parent=None, path="/" 为根；parent=根, path="/文档/" 为一级子文件夹。
    """
    user = models.ForeignKey(User, on_delete=models.CASCADE, related_name='folders')
    name = models.CharField(max_length=255, help_text="文件夹名称")
    parent = models.ForeignKey(
        'self', null=True, blank=True,
        on_delete=models.CASCADE, related_name='children'
    )
    path = models.CharField(
        max_length=2048, db_index=True,
        help_text="物化路径，如 /文档/工作/"
    )
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)

    class Meta:
        verbose_name = "用户文件夹"
        verbose_name_plural = "用户文件夹"
        unique_together = [('user', 'path')]
        ordering = ['path']

    def __str__(self):
        return f"{self.user.username}:{self.path}"

    def save(self, *args, **kwargs):
        """保存时自动计算物化路径"""
        if self.parent:
            self.path = f"{self.parent.path}{self.name}/"
        else:
            self.path = f"/{self.name}/"
        super().save(*args, **kwargs)

    @classmethod
    def ensure_path(cls, user, path: str) -> 'UserFolder':
        """确保路径存在，不存在则递归创建（类似 mkdir -p）"""
        parts = [p for p in path.strip('/').split('/') if p]
        current_parent = None
        current_path = "/"
        folder = None

        for part in parts:
            current_path = f"{current_path}{part}/"
            folder, _ = cls.objects.get_or_create(
                user=user,
                path=current_path,
                defaults={'name': part, 'parent': current_parent}
            )
            current_parent = folder

        return folder
```

### 2.4 UserFile — 文件核心模型

```python
def user_file_upload_to(instance, filename):
    """
    生成用户文件存储路径：user_files/<user_id>/YYYY/MM/<uuid8>_<filename>
    """
    from django.utils import timezone
    import uuid, os
    unique = f"{uuid.uuid4().hex[:8]}_{filename}"
    return timezone.now().strftime(f'user_files/{instance.user_id}/%Y/%m/{unique}')


class UserFile(models.Model):
    """
    用户文件（云盘核心模型）

    设计要点：
    - original_file 保留原始文件，永远不会被修改
    - parsed_markdown 上传时预解析，后续 Agent 调用直接取 MD
    - search_text 是去除 Markdown 格式后的纯文本，用于全文检索
    - file_hash (SHA-256) 用于解析结果复用：同 hash + 未编辑 → 跳过重复解析
    - 存储去重仅限同一文件夹内（同用户+同文件夹+同 hash），不同文件夹允许独立副本
    - 图片文件 parse_status='none'，不做预解析
    - markdown_edited=True 标记用户编辑过 MD，后续不会被自动重解析覆盖
    """

    # ========== 支持的文件类型 ==========
    ALLOWED_MIME_TYPES = {
        # 文档（上传时预解析为 MD）
        'application/pdf': 'document',
        'application/msword': 'document',
        'application/vnd.openxmlformats-officedocument.wordprocessingml.document': 'document',
        'application/vnd.ms-excel': 'document',
        'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet': 'document',
        # 图片（不预解析，后续按需处置）
        'image/jpeg': 'image',
        'image/png': 'image',
        'image/gif': 'image',
        'image/webp': 'image',
    }

    FILE_CATEGORY_CHOICES = [
        ('document', '文档'),
        ('image', '图片'),
    ]

    SOURCE_CHOICES = [
        ('upload', '本地上传'),
        ('url', 'URL上传'),
        ('chat_upload', '聊天上传'),
    ]

    PARSE_STATUS_CHOICES = [
        ('none', '无需解析'),       # 图片
        ('pending', '待解析'),
        ('processing', '解析中'),
        ('completed', '已完成'),
        ('failed', '解析失败'),
    ]

    # ========== 基础信息 ==========
    user = models.ForeignKey(User, on_delete=models.CASCADE, related_name='cloud_files')
    folder = models.ForeignKey(
        UserFolder, null=True, blank=True,
        on_delete=models.SET_NULL, related_name='files'
    )

    # ========== 原始文件 ==========
    original_file = models.FileField(upload_to=user_file_upload_to, help_text="原始文件")
    filename = models.CharField(max_length=512, help_text="用户可见的文件名")
    file_size = models.BigIntegerField(default=0, help_text="文件大小（字节）")
    mime_type = models.CharField(max_length=100, help_text="MIME 类型")
    category = models.CharField(max_length=20, choices=FILE_CATEGORY_CHOICES, help_text="文件分类")
    file_hash = models.CharField(max_length=64, db_index=True, help_text="SHA-256 文件哈希，用于去重")

    # ========== 解析产物（仅文档类型） ==========
    parsed_markdown = models.TextField(blank=True, default='', help_text="预解析的 Markdown 内容")
    parsed_at = models.DateTimeField(null=True, blank=True, help_text="解析完成时间")
    parse_status = models.CharField(
        max_length=20, choices=PARSE_STATUS_CHOICES, default='pending',
        help_text="解析状态"
    )
    parse_source = models.CharField(
        max_length=30, blank=True, default='',
        help_text="解析来源：baidu_cloud / local_fallback / reused / reused_from_attachment / none"
    )
    parse_error = models.TextField(blank=True, default='', help_text="解析错误信息")
    markdown_edited = models.BooleanField(default=False, help_text="用户是否手动编辑过 Markdown")

    # ========== 检索辅助 ==========
    text_preview = models.CharField(max_length=500, blank=True, default='', help_text="前 500 字摘要")
    search_text = models.TextField(blank=True, default='', help_text="去除 MD 格式的纯文本，用于全文检索")

    # ========== 来源追踪 ==========
    source = models.CharField(max_length=20, choices=SOURCE_CHOICES, default='upload', help_text="上传来源")
    source_url = models.URLField(max_length=2048, blank=True, default='', help_text="URL 上传时的原始地址")

    # ========== 软删除 ==========
    is_deleted = models.BooleanField(default=False, help_text="是否已软删除")
    deleted_at = models.DateTimeField(null=True, blank=True)

    # ========== 时间戳 ==========
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)

    class Meta:
        verbose_name = "用户文件"
        verbose_name_plural = "用户文件"
        ordering = ['-created_at']
        indexes = [
            models.Index(fields=['user', 'folder', 'is_deleted']),
            models.Index(fields=['user', 'category', 'is_deleted']),
            models.Index(fields=['user', 'is_deleted', '-created_at']),
            models.Index(fields=['file_hash']),
        ]

    def __str__(self):
        return f"{self.user.username}: {self.filename} ({self.category})"

    @property
    def is_document(self):
        return self.category == 'document'

    @property
    def is_image(self):
        return self.category == 'image'

    @property
    def is_parsed(self):
        return self.parse_status == 'completed'

    @property
    def has_markdown(self):
        return bool(self.parsed_markdown)

    def soft_delete(self):
        """软删除文件并释放配额"""
        from django.utils import timezone
        self.is_deleted = True
        self.deleted_at = timezone.now()
        self.save(update_fields=['is_deleted', 'deleted_at'])
        # 释放配额
        quota = UserStorageQuota.get_or_create_for_user(self.user)
        quota.release(self.file_size)

    def hard_delete(self):
        """物理删除文件和数据库记录"""
        if self.original_file:
            try:
                self.original_file.delete(save=False)
            except Exception:
                pass
        self.delete()

    def to_api_dict(self, include_content=False):
        """转为 API 响应格式"""
        result = {
            'id': self.id,
            'filename': self.filename,
            'file_size': self.file_size,
            'mime_type': self.mime_type,
            'category': self.category,
            'folder_id': self.folder_id,
            'parse_status': self.parse_status,
            'markdown_edited': self.markdown_edited,
            'source': self.source,
            'text_preview': self.text_preview,
            'created_at': self.created_at.isoformat(),
            'updated_at': self.updated_at.isoformat(),
        }
        if self.original_file:
            result['file_url'] = self.original_file.url
        if include_content and self.parsed_markdown:
            result['parsed_markdown'] = self.parsed_markdown
        return result
```

## 3. 文件上传与预解析流程

### 3.1 上传入口与校验

系统存在三个上传入口，最终均汇聚到同一个核心函数：

```
入口 A: /files 管理页上传 ─────────┐
入口 B: /files URL 上传 ───────────┤→ file_service.views_api.upload_files()
入口 C: 聊天页拖拽上传(同步) ──────┘      ↓
                                    _core_upload(user, file, folder, source)
```

`_core_upload()` 校验顺序：

```
1. MIME 白名单检查
   └─ 不在 UserFile.ALLOWED_MIME_TYPES → 400 "不支持的文件类型"

2. 配额检查（UserStorageQuota.can_upload）
   ├─ 单文件大小超限 → 400 "文件大小超过限制（上限 N MB）"
   └─ 总空间不足 → 400 "存储空间不足（剩余 N MB）"

3. 文件名清洗
   └─ 去除路径分隔符、空字节等危险字符；截断至 255 字符

4. SHA-256 计算

5. 同文件夹去重检查
   └─ 同用户 + 同文件夹 + 同 hash + 未删除 → 400 "该文件夹中已存在相同文件"

6. 存储文件

7. 预解析调度（含解析复用）
   └─ 同用户、同 hash、已完成解析且未编辑 → 复用已有 parsed_markdown，跳过实际解析
   └─ 否则 → 执行完整解析流程

8. 更新配额（consume）
9. 返回 UserFile JSON
```

### 3.2 存储路径规则

```
MEDIA_ROOT/
└── user_files/
    └── <user_id>/
        └── YYYY/
            └── MM/
                └── <uuid8>_<原文件名>.<ext>
```

- **用户隔离**：按 `user_id` 分目录，不同用户文件物理隔离
- **UUID 前缀**：避免文件名冲突，同时保留原文件名便于识别
- **按月分层**：避免单目录文件过多
- 与旧的 `media/attachments/` 完全分离，互不干扰

### 3.3 文档预解析调度

`file_service/parser.py` 负责调度，复用现有解析器：

```python
# file_service/parser.py

from agent_service.parsers import parser_factory

# 需要预解析的 MIME 类型
DOCUMENT_MIMES = {
    'application/pdf',
    'application/msword',
    'application/vnd.openxmlformats-officedocument.wordprocessingml.document',
    'application/vnd.ms-excel',
    'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet',
}


def should_preparse(mime_type: str) -> bool:
    """判断该类型是否需要预解析"""
    return mime_type in DOCUMENT_MIMES


def preparse_document(user_file) -> None:
    """
    对文档类文件执行预解析，将结果写入 UserFile。

    流程：
    1. 调用 parser_factory 获取解析器（BaiduDocumentParser 优先 → 本地 fallback）
    2. 解析成功 → 写入 parsed_markdown, search_text, text_preview
    3. 解析失败 → 记录 parse_error，状态设为 failed
    """
    from django.utils import timezone

    user_file.parse_status = 'processing'
    user_file.save(update_fields=['parse_status'])

    parser = parser_factory.get_parser(user_file.mime_type)
    if not parser:
        user_file.parse_status = 'failed'
        user_file.parse_error = f'无可用解析器: {user_file.mime_type}'
        user_file.save(update_fields=['parse_status', 'parse_error'])
        return

    try:
        result = parser.parse(
            user_file.original_file.path,
            mime_type=user_file.mime_type
        )
    except Exception as e:
        user_file.parse_status = 'failed'
        user_file.parse_error = str(e)
        user_file.save(update_fields=['parse_status', 'parse_error'])
        return

    if result.get('success'):
        markdown_text = result.get('text', '')
        plain_text = _strip_markdown(markdown_text)

        user_file.parsed_markdown = markdown_text
        user_file.search_text = plain_text
        user_file.text_preview = plain_text[:500]
        user_file.parse_status = 'completed'
        user_file.parse_source = result.get('source', 'local_fallback')
        user_file.parse_error = ''
        user_file.parsed_at = timezone.now()
    else:
        user_file.parse_status = 'failed'
        user_file.parse_error = result.get('error', '解析失败')

    user_file.save(update_fields=[
        'parsed_markdown', 'search_text', 'text_preview',
        'parse_status', 'parse_source', 'parse_error', 'parsed_at'
    ])


def _strip_markdown(text: str) -> str:
    """去除 Markdown 格式标记，保留纯文本（用于检索索引）"""
    import re
    # 去掉标题标记
    text = re.sub(r'^#{1,6}\s+', '', text, flags=re.MULTILINE)
    # 去掉粗体/斜体
    text = re.sub(r'\*{1,3}(.*?)\*{1,3}', r'\1', text)
    # 去掉链接 [text](url) → text
    text = re.sub(r'\[([^\]]+)\]\([^)]+\)', r'\1', text)
    # 去掉图片 ![alt](url)
    text = re.sub(r'!\[([^\]]*)\]\([^)]+\)', r'\1', text)
    # 去掉表格分隔线
    text = re.sub(r'\|[\s\-:]+\|', '', text)
    # 去掉行首 | 和行尾 |
    text = re.sub(r'^\||\|$', '', text, flags=re.MULTILINE)
    # 去掉代码块标记
    text = re.sub(r'```[\s\S]*?```', '', text)
    text = re.sub(r'`([^`]+)`', r'\1', text)
    # 压缩多余空行
    text = re.sub(r'\n{3,}', '\n\n', text)
    return text.strip()
```

**解析时机**：在 `_core_upload()` 中同步执行。考虑到现有系统已有同步解析实践（`AttachmentHandler._parse_file_attachment` 也是同步的），且百度云文档解析内部自带超时控制，此处保持同步。如后期文件量增大，可改为 Celery 异步任务，模型中已预留 `parse_status` 状态流转。

### 3.4 去重策略（存储去重 + 解析复用）

去重分为两个层面，**存储去重**和**解析复用**，策略不同：

| 层面 | 去重范围 | 行为 |
|------|----------|------|
| **存储去重** | 同用户 + **同文件夹** + 同 hash | 拒绝上传，返回 400 |
| **解析复用** | 同用户 + 同 hash（跨文件夹） | 允许上传（独立存储），但复用已有的 MD 解析结果 |

#### 存储工具

```python
# file_service/storage.py

import hashlib

def compute_file_hash(file_obj) -> str:
    """
    计算上传文件的 SHA-256 哈希值。
    file_obj: Django UploadedFile 或类似可 read/seek 的对象。
    """
    sha256 = hashlib.sha256()
    file_obj.seek(0)
    for chunk in file_obj.chunks(chunk_size=8192):
        sha256.update(chunk)
    file_obj.seek(0)  # 重置指针，后续还要保存文件
    return sha256.hexdigest()
```

#### 存储去重：同文件夹内禁止重复

```python
file_hash = compute_file_hash(uploaded_file)

# 同用户 + 同文件夹 + 同 hash + 未删除 → 拒绝
duplicate_in_folder = UserFile.objects.filter(
    user=user, folder=target_folder, file_hash=file_hash, is_deleted=False
).exists()

if duplicate_in_folder:
    return Response(
        {"error": "该文件夹中已存在相同文件"},
        status=400
    )
```

不同文件夹上传同一文件（同 hash）是被允许的，每份独立存储、独立管理、独立计入配额。这与操作系统文件管理器的行为一致。

#### 解析复用：跨文件夹复用已有 MD

当文件通过存储去重检查后（即允许上传），在预解析阶段检查是否有可复用的解析结果：

```python
# file_service/parser.py — preparse_document() 起始处

def preparse_document(user_file) -> None:
    # ---- 解析复用检查 ----
    # 查找同用户、同 hash、已完成解析且未手动编辑的文件
    donor = UserFile.objects.filter(
        user=user_file.user,
        file_hash=user_file.file_hash,
        parse_status='completed',
        markdown_edited=False,
        is_deleted=False,
    ).exclude(id=user_file.id).first()

    if donor and donor.parsed_markdown:
        # 复用已有解析结果，跳过实际解析
        user_file.parsed_markdown = donor.parsed_markdown
        user_file.search_text = donor.search_text
        user_file.text_preview = donor.text_preview
        user_file.parse_status = 'completed'
        user_file.parse_source = 'reused'  # 标记来源为复用
        user_file.parse_error = ''
        user_file.parsed_at = timezone.now()
        user_file.save(update_fields=[...])
        return

    # ---- 无可复用结果，执行完整解析 ----
    # ...（原有解析流程不变）
```

**复用条件**（必须同时满足）：

| 条件 | 说明 |
|------|------|
| 同用户 | 不跨用户复用（数据隔离） |
| 同 `file_hash` | SHA-256 完全一致 |
| `parse_status = 'completed'` | 供体文件解析成功 |
| `markdown_edited = False` | 供体的 MD 未被用户手动编辑（编辑后的内容不代表原文件的自动解析结果） |
| `is_deleted = False` | 供体未被删除 |

**复用标记**：`parse_source = 'reused'`，与 `'baidu_cloud'` / `'local_fallback'` 区分，方便调试和统计。

**边界情况**：

| 场景 | 行为 |
|------|------|
| 同文件夹上传相同文件 | 存储去重拒绝，返回 400 |
| 不同文件夹上传相同文件 | 允许上传，复用 MD 解析，独立存储 |
| 供体的 MD 已被用户编辑 | 不复用（编辑内容是用户个性化的），执行完整解析 |
| 供体已被软删除 | 不复用，执行完整解析 |
| 多个供体候选（多份相同文件） | 取第一个匹配的（`first()`），任意一份即可 |
| 用户删除供体后 | 已复用的文件不受影响（MD 内容已独立保存在各自记录中） |

### 3.5 解析器复用策略

`file_service` **不创建新的解析器**，而是直接 import 现有模块：

```python
from agent_service.parsers import parser_factory         # 工厂单例
from agent_service.parsers.document_parser import (      # 按需直接引用
    BaiduDocumentParser, PDFParser, WordParser, ExcelParser
)
```

解析器保留在 `agent_service/parsers/` 的原因：

1. `agent_service` 的 `AttachmentHandler` 仍需使用（图片 Base64、内部元素解析等）
2. 解析器本身不依赖 `agent_service` 的业务逻辑，仅依赖外部库和配置
3. 避免循环依赖：`file_service` → `agent_service.parsers`（单向）

后期如需进一步解耦，可将 `parsers/` 提升为独立的顶层 Python 包。当前阶段保持最小改动。

## 5. URL 上传与白名单机制

### 5.1 安全威胁分析

URL 上传本质是服务端代替用户发起 HTTP 请求下载文件，主要风险：

| 风险 | 说明 | 对策 |
|------|------|------|
| **SSRF** | 攻击者提交内网 URL（`http://127.0.0.1`、`http://192.168.x.x`）访问内部服务 | 禁止内网/回环/链路本地地址；DNS 解析后二次检查 IP |
| **资源耗尽** | 下载超大文件或极慢连接占用服务器资源 | 文件大小限制（配额）+ 下载超时（30s）+ 流式下载限速 |
| **恶意内容** | 下载非预期文件类型（可执行文件、脚本等） | Content-Type 白名单校验 + 文件头魔数验证 |
| **开放重定向** | URL 经多次 302 跳转最终指向内网 | 禁止重定向或限制重定向跳数（最多 3 次） |

### 5.2 白名单策略

`file_service/url_fetcher.py` 中实现多层防护：

```python
# ========== 域名白名单 ==========
# 默认白名单（可在 settings.py 中覆盖为 FILE_SERVICE_URL_WHITELIST）
DEFAULT_URL_DOMAIN_WHITELIST = [
    # 网盘直链
    'aliyundrive.com',
    'lanzou.com', 'lanzoui.com', 'lanzoux.com',
    'weiyun.com',
    # 开发者资源
    'raw.githubusercontent.com',
    'gist.githubusercontent.com',
    'objects.githubusercontent.com',
    # 文件分享服务
    'dl.dropboxusercontent.com',
    'drive.google.com',
    # 用户可在 settings.py 追加
]

# ========== 禁止的 IP/网段 ==========
BLOCKED_IP_RANGES = [
    # 回环
    ipaddress.ip_network('127.0.0.0/8'),
    ipaddress.ip_network('::1/128'),
    # 内网
    ipaddress.ip_network('10.0.0.0/8'),
    ipaddress.ip_network('172.16.0.0/12'),
    ipaddress.ip_network('192.168.0.0/16'),
    # 链路本地
    ipaddress.ip_network('169.254.0.0/16'),
    ipaddress.ip_network('fe80::/10'),
    # 元数据服务（云厂商）
    ipaddress.ip_network('169.254.169.254/32'),
]

# ========== Content-Type 白名单 ==========
ALLOWED_CONTENT_TYPES = set(UserFile.ALLOWED_MIME_TYPES.keys())
```

**域名匹配逻辑**：

```python
def is_domain_allowed(url: str) -> bool:
    """检查 URL 域名是否在白名单中（支持子域名匹配）"""
    from urllib.parse import urlparse
    hostname = urlparse(url).hostname
    if not hostname:
        return False

    whitelist = getattr(settings, 'FILE_SERVICE_URL_WHITELIST', DEFAULT_URL_DOMAIN_WHITELIST)
    for allowed in whitelist:
        if hostname == allowed or hostname.endswith('.' + allowed):
            return True
    return False
```

### 5.3 下载流程

```python
def fetch_url(url: str, user) -> dict:
    """
    从 URL 下载文件，返回临时文件对象。

    流程：
    1. URL 格式校验 + 协议校验（仅 http/https）
    2. 域名白名单检查
    3. DNS 解析 → IP 黑名单检查（防 SSRF）
    4. HEAD 请求 → Content-Type/Content-Length 预检
    5. GET 流式下载 → 写入临时文件
    6. 返回 {success, temp_file, filename, mime_type, file_size}

    Returns:
        {
            "success": bool,
            "temp_file": TemporaryUploadedFile | None,
            "filename": str,
            "mime_type": str,
            "file_size": int,
            "error": str
        }
    """
```

完整流程伪代码：

```
1. 解析 URL → 仅允许 http:// 和 https://
2. 域名白名单检查 → 不在白名单 → 拒绝
3. socket.getaddrinfo(hostname) → 获取真实 IP
   └─ IP 在 BLOCKED_IP_RANGES → 拒绝（SSRF 防护）
4. HEAD 请求（timeout=10s, allow_redirects=True, max_redirects=3）
   ├─ Content-Type 不在白名单 → 拒绝
   └─ Content-Length > quota.max_file_size → 拒绝
5. GET 流式下载（timeout=30s, stream=True）
   ├─ 累计下载量超过 max_file_size → 中止
   └─ 写入 Django TemporaryUploadedFile
6. 从 Content-Disposition 或 URL 路径提取文件名
7. 返回结果
```

**API 端点**：

```
POST /api/files/upload-url/
Body: {"url": "https://example.com/file.pdf", "folder_id": 3}
```

后续流程与普通上传一致（配额检查 → 去重 → 存储 → 预解析）。

### 6.1 编辑流程


```python
@api_view(['GET', 'PUT'])
@permission_classes([IsAuthenticated])
def file_markdown(request, file_id):
    """
    GET  /api/files/<id>/markdown/  — 读取 Markdown 内容
    PUT  /api/files/<id>/markdown/  — 更新 Markdown 内容
    """
    user_file = get_object_or_404(UserFile, id=file_id, user=request.user, is_deleted=False)

    if not user_file.is_document:
        return Response({"error": "仅文档类文件支持 Markdown 操作"}, status=400)

    if request.method == 'GET':
        return Response({
            "id": user_file.id,
            "filename": user_file.filename,
            "parsed_markdown": user_file.parsed_markdown,
            "markdown_edited": user_file.markdown_edited,
            "parse_status": user_file.parse_status,
        })

    # PUT: 更新
    content = request.data.get('content', '')
    if not content.strip():
        return Response({"error": "内容不能为空"}, status=400)

    plain_text = _strip_markdown(content)

    user_file.parsed_markdown = content
    user_file.search_text = plain_text
    user_file.text_preview = plain_text[:500]
    user_file.markdown_edited = True
    user_file.save(update_fields=[
        'parsed_markdown', 'search_text', 'text_preview', 'markdown_edited', 'updated_at'
    ])

    return Response({
        "id": user_file.id,
        "message": "保存成功",
        "markdown_edited": True,
    })
```

### 6.2 编辑与原文件的关系

| 操作 | `original_file` | `parsed_markdown` | `markdown_edited` |
|------|-----------------|--------------------|--------------------|
| 上传后自动解析 | 保存原文件 | 自动生成 | `False` |
| 用户编辑 MD | **不变** | 更新为用户版本 | `True` |
| 下载原格式 | 返回 original_file | — | — |
| 下载 MD 格式 | — | 返回 parsed_markdown | — |
| 重新解析（管理员操作） | 不变 | 仅 `markdown_edited=False` 时覆盖 | 不变 |

**核心原则**：
- 原始文件始终不被修改，保证用户可以随时下载回原始格式
- 用户编辑的 Markdown 与原文件解耦，`markdown_edited=True` 作为保护标记
- 搜索索引（`search_text`）始终跟随 `parsed_markdown` 同步更新

```python
# file_service/sync.py

def sync_chat_upload_to_cloud(user, uploaded_file, session_attachment) -> UserFile:
    """
    将聊天上传的文件同步到云盘默认路径 /聊天上传/

    流程：
    1. 确保 /聊天上传/ 文件夹存在（UserFolder.ensure_path）
    2. 计算 file_hash → 检查去重
    3. 创建 UserFile（复用 SessionAttachment 的解析结果）
    4. 回填 SessionAttachment.cloud_file
    5. 更新配额
    """
    from file_service.models import UserFile, UserFolder, UserStorageQuota
    from file_service.storage import compute_file_hash

    # 确保默认文件夹
    folder = UserFolder.ensure_path(user, '/聊天上传/')

    # 去重检查：同文件夹内禁止重复
    file_hash = compute_file_hash(uploaded_file)
    existing_in_folder = UserFile.objects.filter(
        user=user, folder=folder, file_hash=file_hash, is_deleted=False
    ).first()

    if existing_in_folder:
        # 同文件夹下已有相同文件，直接关联，不重复创建
        session_attachment.cloud_file = existing_in_folder
        session_attachment.save(update_fields=['cloud_file'])
        return existing_in_folder

    # 创建 UserFile
    mime_type = uploaded_file.content_type or ''
    category = UserFile.ALLOWED_MIME_TYPES.get(mime_type, 'document')

    user_file = UserFile(
        user=user,
        folder=folder,
        original_file=uploaded_file,
        filename=uploaded_file.name or 'untitled',
        file_size=uploaded_file.size or 0,
        mime_type=mime_type,
        category=category,
        file_hash=file_hash,
        source='chat_upload',
    )

    # 解析结果复用策略（优先级从高到低）：
    # 1. 从 SessionAttachment 复用（当次聊天刚解析过）
    # 2. 从云盘中同 hash 已解析文件复用
    # 3. 标记为 pending，由后续流程解析
    if category == 'image':
        user_file.parse_status = 'none'
    elif session_attachment.parsed_text:
        # 优先复用 SessionAttachment 的解析结果
        user_file.parsed_markdown = session_attachment.parsed_text
        user_file.search_text = _strip_markdown(session_attachment.parsed_text)
        user_file.text_preview = user_file.search_text[:500]
        user_file.parse_status = 'completed'
        user_file.parse_source = 'reused_from_attachment'
    else:
        # 尝试从云盘中同 hash 已解析且未编辑的文件复用
        donor = UserFile.objects.filter(
            user=user, file_hash=file_hash,
            parse_status='completed', markdown_edited=False, is_deleted=False,
        ).first()
        if donor and donor.parsed_markdown:
            user_file.parsed_markdown = donor.parsed_markdown
            user_file.search_text = donor.search_text
            user_file.text_preview = donor.text_preview
            user_file.parse_status = 'completed'
            user_file.parse_source = 'reused'
        else:
            user_file.parse_status = 'pending'

    user_file.save()

    # 关联
    session_attachment.cloud_file = user_file
    session_attachment.save(update_fields=['cloud_file'])

    # 更新配额
    quota = UserStorageQuota.get_or_create_for_user(user)
    quota.consume(user_file.file_size)

    return user_file
```

```python
    """
    从云盘文件创建 SessionAttachment（不重复上传/解析）
```

### 8.1 基础文本匹配方案

当前阶段不引入向量化，使用基于 `search_text` 字段的多策略文本匹配：

```python
# file_service/search.py

class FileSearchEngine:
    """基础文本检索引擎"""

    @staticmethod
    def search(user, query: str, limit: int = 10,
               category: str = None, folder_id: int = None) -> list:
        """
        在用户云盘文件中搜索相关内容。

        评分策略（加权叠加）：
        1. 精确短语匹配      +10 分
        2. 所有关键词 AND 命中  +5 分
        3. 单个关键词命中      +1 分/个
        4. 文件名命中          +3 分

        返回: [{file: UserFile, score: int, snippet: str}, ...]
        """
        from file_service.models import UserFile

        qs = UserFile.objects.filter(user=user, is_deleted=False)

        # 仅搜索有文本内容的文件
        qs = qs.exclude(search_text='')

        if category:
            qs = qs.filter(category=category)
        if folder_id:
            qs = qs.filter(folder_id=folder_id)

        keywords = query.split()
        if not keywords:
            return []

        results = []
        q_lower = query.lower()

        for uf in qs.iterator():
            text = uf.search_text.lower()
            filename = uf.filename.lower()
            score = 0

            # 精确短语匹配
            if q_lower in text:
                score += 10

            # 文件名包含查询词
            if q_lower in filename:
                score += 3

            # 所有关键词命中
            hits = [kw.lower() for kw in keywords if kw.lower() in text]
            if len(hits) == len(keywords):
                score += 5

            # 单个关键词计分
            score += len(hits)

            if score > 0:
                snippet = FileSearchEngine._extract_snippet(text, keywords)
                results.append({
                    'file': uf,
                    'score': score,
                    'snippet': snippet,
                })

        results.sort(key=lambda x: -x['score'])
        return results[:limit]

    @staticmethod
    def _extract_snippet(text: str, keywords: list, context_chars: int = 120) -> str:
        """
        从文本中提取包含关键词的上下文片段。
        找到第一个关键词出现的位置，截取前后各 context_chars 个字符。
        """
        text_lower = text.lower()
        best_pos = -1

        for kw in keywords:
            pos = text_lower.find(kw.lower())
            if pos >= 0:
                best_pos = pos
                break

        if best_pos < 0:
            return text[:context_chars * 2] + '...' if len(text) > context_chars * 2 else text

        start = max(0, best_pos - context_chars)
        end = min(len(text), best_pos + context_chars)
        snippet = text[start:end]

        if start > 0:
            snippet = '...' + snippet
        if end < len(text):
            snippet = snippet + '...'

        return snippet
```

```python
# file_service/search.py

from abc import ABC, abstractmethod
from typing import List, Dict, Any, Optional


class RAGSearchProvider(ABC):
    """
    RAG 检索提供者抽象接口

    所有实现必须提供文档索引、语义检索、文档删除三个核心能力。
    """

    @abstractmethod
    def index_document(self, doc_id: str, text: str,
                       metadata: Optional[Dict] = None) -> bool:
        """
        将文档索引到知识库。

        Args:
            doc_id: 唯一文档标识（通常为 UserFile.id 的字符串形式）
            text: 文档全文（parsed_markdown 或 search_text）
            metadata: 附加元数据（文件名、类型、用户 ID 等）

        Returns:
            是否成功
        """
        pass

    @abstractmethod
    def search(self, query: str, user_id: int,
               top_k: int = 5, filters: Optional[Dict] = None) -> List[Dict[str, Any]]:
        """
        语义检索。

        Args:
            query: 查询文本
            user_id: 用户 ID（数据隔离）
            top_k: 返回最相关的前 K 条
            filters: 可选过滤条件（category, folder_id 等）

        Returns:
            [{"doc_id": str, "score": float, "chunk_text": str, "metadata": dict}, ...]
        """
        pass

    @abstractmethod
    def delete_document(self, doc_id: str) -> bool:
        """从知识库中删除文档索引"""
        pass

    def batch_index(self, documents: List[Dict]) -> Dict[str, bool]:
        """
        批量索引（默认逐个调用，子类可优化为批量 API）。

        Args:
            documents: [{"doc_id": str, "text": str, "metadata": dict}, ...]

        Returns:
            {"doc_id": success_bool, ...}
        """
        results = {}
        for doc in documents:
            results[doc['doc_id']] = self.index_document(
                doc['doc_id'], doc['text'], doc.get('metadata')
            )
        return results


class LocalTextSearchProvider(RAGSearchProvider):
    """
    本地文本检索实现（当前默认）。
    index_document / delete_document 为空操作（不需要显式索引，直接查库）。
    search 委托给 FileSearchEngine。
    """

    def index_document(self, doc_id, text, metadata=None):
        # 本地方案不需要显式索引，search_text 字段已在上传时填充
        return True

    def search(self, query, user_id, top_k=5, filters=None):
        from django.contrib.auth.models import User
        user = User.objects.get(id=user_id)
        results = FileSearchEngine.search(
            user=user, query=query, limit=top_k,
            category=filters.get('category') if filters else None,
            folder_id=filters.get('folder_id') if filters else None,
        )
        return [{
            'doc_id': str(r['file'].id),
            'score': r['score'],
            'chunk_text': r['snippet'],
            'metadata': {
                'filename': r['file'].filename,
                'category': r['file'].category,
            }
        } for r in results]

    def delete_document(self, doc_id):
        return True

def get_search_provider() -> RAGSearchProvider:
    """
    获取当前激活的检索提供者。

    - 'local'（默认）
    - 'aliyun'
```

```python
from langchain_core.tools import tool


@tool
def search_user_files(query: str, limit: int = 5) -> str:
    """
    在用户的云盘文件中搜索相关内容。

    根据搜索词在用户已上传的文档中查找相关内容片段。
    适用于用户询问"我之前上传的XX文件里说了什么"等场景。

    Args:
        query: 搜索关键词
        limit: 返回结果数量上限，默认 5

        匹配的文件列表和内容摘选

    # user 从 tool 运行时上下文注入（与现有工具一致）
    import threading
    user = getattr(threading.current_thread(), '_current_user', None)
    if not user:
        return "无法确定当前用户"

    from file_service.search import FileSearchEngine
    results = FileSearchEngine.search(user=user, query=query, limit=limit)

    if not results:
        return f"未找到与「{query}」相关的文件内容。"

    lines = [f"找到 {len(results)} 个相关结果：\n"]
    for i, r in enumerate(results, 1):
        uf = r['file']
        lines.append(f"**#{i} {uf.filename}**")
        lines.append(f"  相关片段：{r['snippet']}")
        lines.append("")

    return '\n'.join(lines)
```

## 9. 认证与权限体系

### 9.1 页面认证（Session）

`/files` 管理页仅需 Django Session 认证，与 `/home` 一致：

```python
# file_service/views_page.py

from django.contrib.auth.decorators import login_required

@login_required
def files_page(request):
    """云盘管理页"""
    return render(request, 'file_service/files.html')
```

未登录用户访问 `/files` 自动重定向到登录页。

### 9.2 API 双认证（Session + Token）

所有 `/api/files/` 端点均同时支持 Session 认证和 Token 认证，以便：
- **浏览器前端**（/files 页面、/home 聊天页）使用 Session Cookie
- **外部 API 客户端**（自动化脚本、第三方集成）使用 Token Header

实现方式：复用项目现有的 DRF 认证配置。

```python
# file_service/views_api.py

from rest_framework.decorators import api_view, permission_classes
from rest_framework.permissions import IsAuthenticated

# DRF 已在 settings.py 中配置了 SessionAuthentication + TokenAuthentication
# 所有使用 @permission_classes([IsAuthenticated]) 的视图自动支持两种认证

@api_view(['GET'])
@permission_classes([IsAuthenticated])
def list_files(request):
    """列出用户文件"""
    user = request.user  # 无论 Session 还是 Token，都能获取到 user
    ...
```

现有 `settings.py` DRF 认证配置（确认已包含）：

```python
REST_FRAMEWORK = {
    'DEFAULT_AUTHENTICATION_CLASSES': [
        'rest_framework.authentication.SessionAuthentication',
        'rest_framework.authentication.TokenAuthentication',
    ],
    # ...
}
```

### 9.3 文件访问权限

**核心规则**：每个用户只能访问自己的文件和文件夹，不可跨用户。

```python
# 所有涉及单个文件操作的视图都必须验证 user 归属
user_file = get_object_or_404(UserFile, id=file_id, user=request.user, is_deleted=False)
```

**文件下载鉴权**：

```python
@api_view(['GET'])
@permission_classes([IsAuthenticated])
def download_file(request, file_id):
    """下载原始文件"""
    user_file = get_object_or_404(UserFile, id=file_id, user=request.user, is_deleted=False)

    response = FileResponse(
        user_file.original_file.open('rb'),
        content_type=user_file.mime_type,
    )
    # 使用 RFC 5987 编码文件名，支持中文
    response['Content-Disposition'] = (
        f"attachment; filename*=UTF-8''{quote(user_file.filename)}"
    )
    return response
```

**静态文件服务限制**：

`user_files/` 目录虽然在 `MEDIA_ROOT` 下，但生产环境中不应直接通过 Nginx/静态文件服务暴露。所有文件下载必须经过上述鉴权视图，可选方案：

- 开发环境：Django `FileResponse` 直接服务（当前方案，可接受）
- 生产环境（后期）：Nginx `X-Accel-Redirect` 或签名 URL
