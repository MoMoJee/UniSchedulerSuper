# 日历触屏滑动功能 - 快速测试指南

## 🎯 测试目标
验证日历触屏滑动功能是否正常工作

## 📱 前置条件
- 使用支持触摸的设备（手机、平板、触摸屏笔记本）
- 或使用浏览器的设备模拟模式（Chrome DevTools）

## 🧪 测试步骤

### 1. 启动应用
```bash
cd d:\PROJECTS\UniSchedulerSuper
python manage.py runserver
```

### 2. 打开浏览器
- 访问: http://localhost:8000/home/
- 登录你的账号

### 3. 启用移动设备模拟（桌面测试）
如果你使用桌面浏览器：
1. 按 F12 打开开发者工具
2. 点击工具栏的设备切换按钮（Ctrl+Shift+M）
3. 选择一个移动设备（如 iPhone 12 Pro）

### 4. 基础滑动测试

#### 测试 1: 向右滑动（上一个时间段）
1. 在日历区域向右滑动
2. **预期结果**: 
   - 日历切换到上一个月/周/日
   - 屏幕左侧短暂显示向左箭头指示器
   - 控制台输出: `🔄 向右滑动，切换到上一个时间段`

#### 测试 2: 向左滑动（下一个时间段）
1. 在日历区域向左滑动
2. **预期结果**:
   - 日历切换到下一个月/周/日
   - 屏幕右侧短暂显示向右箭头指示器
   - 控制台输出: `🔄 向左滑动，切换到下一个时间段`

#### 测试 3: 短距离滑动
1. 在日历上快速短距离滑动（小于50像素）
2. **预期结果**: 不触发切换

#### 测试 4: 垂直滑动
1. 在日历上上下滑动
2. **预期结果**: 
   - 页面正常滚动
   - 不触发日历切换

### 5. 区域判断测试

#### 测试 5: 工具栏区域
1. 在顶部工具栏（包含"今天"、"周"、"月"等按钮的区域）滑动
2. **预期结果**: 不触发日历切换

#### 测试 6: 筛选按钮
1. 在"筛选"按钮上滑动
2. **预期结果**: 不触发日历切换

#### 测试 7: 事件卡片
1. 在日历的事件卡片上滑动
2. **预期结果**: 
   - 如果是水平滑动，触发日历切换
   - 这是正常行为，用户可以在事件上滑动来浏览

### 6. 多视图测试

#### 测试 8: 月视图
1. 切换到月视图（点击"月"按钮）
2. 在日历上左右滑动
3. **预期结果**: 切换到上/下一个月

#### 测试 9: 周视图
1. 切换到周视图（点击"周"按钮）
2. 在日历上左右滑动
3. **预期结果**: 切换到上/下一周

#### 测试 10: 日视图
1. 切换到2日视图（点击"2日"按钮）
2. 在日历上左右滑动
3. **预期结果**: 切换到上/下一天

#### 测试 11: 列表视图
1. 切换到列表视图（点击"列表"按钮）
2. 在日历上左右滑动
3. **预期结果**: 切换到上/下一周的列表

### 7. 边界情况测试

#### 测试 12: 斜向滑动
1. 在日历上以45度角滑动
2. **预期结果**: 
   - 如果水平距离大于垂直距离，可能触发切换
   - 如果垂直距离更大，应该是正常滚动

#### 测试 13: 快速连续滑动
1. 快速连续向左滑动3次
2. **预期结果**: 
   - 日历切换3次
   - 每次都显示指示器

## ✅ 验收标准

所有测试都应该通过，具体要求：
- ✅ 左右滑动能正确切换日历时间段
- ✅ 滑动指示器正确显示
- ✅ 工具栏和按钮区域不触发滑动
- ✅ 垂直滚动不受影响
- ✅ 所有视图类型都支持滑动
- ✅ 控制台没有错误信息

## 🐛 常见问题

### 问题 1: 滑动没有反应
**检查项**:
- 是否使用了触摸设备或开启了设备模拟？
- 控制台是否有错误？
- 滑动距离是否足够（至少50像素）？

### 问题 2: 滑动与滚动冲突
**检查项**:
- 是否在时间网格区域滑动？（这个区域需要滚动）
- 尝试在日历顶部（日期标题区域）滑动

### 问题 3: 指示器不显示
**检查项**:
- 检查 CSS 是否正确加载
- 检查控制台是否有相关错误
- 指示器显示时间很短（0.6秒），可能需要仔细观察
