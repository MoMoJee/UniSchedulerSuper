# FR-UI Apple 交互与工作台深度优化验收报告

日期：2026-07-18  
状态：通过

## 1. 本轮目标

本轮不是对十二条反馈逐项打补丁，而是以旧版工作台的信息密度、功能完整性和 Apple Design 的层级、克制、直接操控原则为基线，重新检查主页、日历、待办、提醒、Agent、附件、文件、分享组、设置和主题之间的整体关系。

验收重点包括：模块边界清晰、日历成为视觉主体、控件不重复、异步刷新不破坏当前界面、颜色具有信息含义、弹窗与设置符合统一交互模型，并保证所有修改仍走既有 V2 与专用业务接口。

## 2. 已完成的界面与交互升级

### 2.1 主工作台与视觉层级

- 待办、提醒、日历和智能助手成为四个独立悬浮圆角材料面板，使用统一边框、阴影、圆角和内边距。
- 顶部全局搜索继续承担“灵动岛”入口，设置、菜单和账户信息保持低干扰。
- 日历工具区压缩为一行，删除可见的“日程工作区”“我的日程”冗余标题，并保留屏幕阅读器语义标题。
- 合并重复的日程组入口，只保留一个“日程组”管理按钮。
- 金色主题增加面板描边、当前视图、主操作和顶部岛屿装饰，同时保持文字对比度。
- 常驻滚动条改为细轨，接近或悬停时才增强可见性。

### 2.2 日历工作区

- `today/month/week/list` 完整汉化为“今天/月/周/列表”，并保留旧版 2 日视图。
- 日程继续按所属日程组显示颜色，不再统一为蓝色。
- 个人日程若分享至一个或多个分享组，会在事件上显示对应主题色圆点。
- “我的日程/分享组”全部保持轮廓按钮形态，仅当前选择填充；超宽时支持鼠标滚轮横向浏览，不显示固定横向滚动条。
- 保留拖拽框选时间创建日程，以及事件拖拽、缩放和重复日程范围编辑入口。
- 日历异步加载时保持 FullCalendar 实例挂载，仅显示轻量更新提示；数据到达后增量替换事件，避免整个日历闪烁。
- 日期、视图、日程组和筛选条件继续写入 URL，以支持刷新恢复。
- 恢复“当前学期 · 第 X 周”显示，并可直接进入日程偏好设置。

### 2.3 待办与提醒

- 待办恢复列表/四象限两种视图、筛选、创建、完成、编辑、删除和转日程入口。
- 四象限按重要/紧急组合严格分类，缺少分类字段的记录进入独立“其他”区域，不再错误混入“不重要不紧急”。
- 宽布局采用四象限矩阵并将“其他”横跨底部；窄布局自适应为较少列数，避免文字与滚动内容重叠。
- 待办卡片通过所属日程组颜色的边线与圆点表达归属；创建和编辑支持选择日程组。
- 提醒卡片使用分组颜色，并保持筛选、创建、查看与系列编辑入口。

### 2.4 Agent 与附件

- 去除大块空白和含义不清的“会话上下文与任务”，改为紧凑状态条：上下文占用、任务状态和“整理记忆”。
- 空会话提供三条可直接填入输入框的建议，消息区与输入区始终占据正确网格行。
- 附件入口改为居中模态框，分为“系统内容 / 我的文件 / 从设备上传”。
- 本地文件输入不再暴露浏览器原生控件，改为拖放区域和明确的“选择文件”按钮。
- 云盘文件选择也使用居中子模态框，修复父级侧拉层拦截指针事件的问题。
- Agent 会话、工具选择、Quick Action、回滚后附件恢复、内部 Event/Todo/Reminder 附件和本地文件附件的原合同保持不变。

### 2.5 设置、分享与文件

- 设置采用单一居中设置中心和分段分类，不再跳转到第二套页面。
- 基本设置增加“管理分享组”入口；分享组仍通过统一工作台 surface 管理。
- 日程偏好恢复学期起始日编辑器，支持新增、删除、修改名称、年、月、日，并驱动主页周次显示。
- 主题与金色装饰同时从服务端偏好初始化，保存、取消和刷新后的状态一致。
- 文件管理统一工具栏、面包屑、文件夹树、列表/网格切换、文件卡片和预览/重命名/移动/删除/URL 上传操作的布局。
- 文件选择器与文件管理主界面的弹层模式一致，避免多套交互语言。

## 3. 独立审查发现并修复的问题

### 3.1 渲染阶段导航

FullCalendar 的 `datesSet` 会在子树挂载过程中执行。旧实现此时直接调用 React Router 写 URL，产生“应在 effect 中调用 navigate”的警告，并可能造成初次加载抖动。现在回调只记录可见范围，URL 持久化在挂载后的 effect 中执行。

### 3.2 编辑器重复 key

日程编辑器与提醒编辑器是同级组件，选中日程时二者曾同时使用 `event-1` 一类实体 ID 作为 key。React 因此不能稳定识别组件身份。现在分别使用 `event-editor-*` 与 `reminder-editor-*` 命名空间。

### 3.3 重复实例渲染身份

同一重复定义可投影多个实例，不能只以定义 ID 作为 FullCalendar 事件 ID。当前使用“实体类型 + 定义 ID + recurrence_id/开始时间”生成实例级 ID。

### 3.4 分享标识数据缺口

个人 occurrence 原先没有携带分享组关系，前端无法画出分享圆点。V2 occurrence 现在返回 `share_group_ids`，前端 mapper、类型和日历渲染已完整贯通。

### 3.5 待办裁切、三栏错位与周视图信息密度

待办板原先将四象限和“其他”同时压缩到固定高度，导致“其他”实际内容被裁切，外层 `scrollHeight` 又等于 `clientHeight`，鼠标滚轮没有可滚动距离。当前四象限固定为内部独立滚动区，“其他”按内容高度参与外层布局，待办板统一承担纵向滚动，避免卡片越界或重叠。

主页中栏原先沿用普通页面的 `md:p-6`，比左右浮动面板多 24 px 内缩。主页现使用统一 8 px 外边距，左栏、日历和 Agent 的顶部与底部坐标一致。

周视图事件原先强制 `white-space: nowrap` 和省略号。当前仅在 time-grid 视图中将时间与标题分行，标题依据事件高度最多显示五行；月视图继续单行省略，避免高密度月份网格被撑开。
