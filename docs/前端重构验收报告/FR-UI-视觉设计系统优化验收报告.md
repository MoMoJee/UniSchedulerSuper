# FR-UI 视觉设计系统优化验收报告

> 日期：2026-07-18  
> 结论：在不改变现有业务信息架构和交互能力的前提下，完成 Home、Planner、Agent、全局搜索、设置、文件管理与分享组的统一视觉升级，并修复视觉审查中暴露的结构性排版错误。

## 设计目标与依据

本轮不再以零散页面“逐个调 CSS”，而是先建立统一的视觉契约，再让各功能模块复用。参考 Apple Design skill 中关于即时反馈、材质层级、空间连续性、克制动效及无障碍降级的建议，落地以下规则：

- 使用统一画布、表面、边界、强调色、圆角和阴影 token，消除各模块各自定义一套样式造成的割裂。
- 顶部灵动岛、中央弹层和浮动菜单采用有边界的半透明材质；关闭透明效果时自动退化为不透明表面。
- 按钮按下立即缩放反馈，弹层采用短促的 materialize 动画；`prefers-reduced-motion` 下关闭非必要运动。
- 内容层级通过字号、字重、间距和表面深度表达，不依靠大量高饱和色块。
- 高对比度、减少透明效果和减少动态效果均有系统偏好适配。

## 实际修复

### 全局视觉系统

- 新增 `visual-polish.css`，集中维护画布、表面、描边、阴影、圆角、按钮状态、焦点环、动效及可访问性降级。
- 更新通用 Button 变体，统一主按钮、次按钮、危险按钮与幽灵按钮的高度、边界、阴影和按压反馈。
- 中央弹层增加柔和背景模糊、独立内容材质、稳定的标题/正文/底栏层级与动态视口高度限制。

### Home、Planner 与 Agent

- 修复 Planner 网格行定义错误。工具栏、筛选区、范围导航和日历现在拥有明确的四行结构，日历占据剩余空间，不再出现大段无意义空白。
- 调整工作区标题、操作按钮、范围导航、FullCalendar 工具栏、日期单元和事件视觉，使日历重新成为主视觉焦点，并保留日程组颜色。
- 待办四象限、提醒区、Agent 标题、消息卡片和输入区统一表面、分隔与 hover/press 状态。

### 搜索、设置、文件与分享组

- 全局搜索改为明确的搜索主输入、类型 chips、进阶过滤卡和结果区层级。
- 修复设置页 CSS Module 隔离错误：嵌入弹层时只展示当前分类，不再把全部设置面板堆叠在同一屏；AI 设置复选框、字段网格和保存动作恢复正确对齐。
- 修复文件列表选择器误命中内容按钮的问题，文件名不再被操作列挤成逐字竖排。
- 修复文件管理嵌入根网格未约束高度的问题，标题区、侧栏、工具栏和文件列表在弹层内稳定占位，不再出现巨大空白。
- 分享组使用两列响应式卡片，正文与操作区分层；卡片保留组颜色、角色、成员数、ID 和全部管理动作。
- 修复 Radix Select 的 portal 层级低于中央弹层遮罩、导致重复规则等下拉项可见但偶发无法点击的问题。
- 修复全屏文件管理与应用顶栏产生重复 `banner` landmark 的无障碍问题。
- 为 FullCalendar 投影实例生成“类型 + 实体 + recurrence/start”的稳定渲染 ID，避免重复系列实例共享实体 ID。

## 浏览器实测

使用 MoMoJee 已登录真实会话逐页检查：

- Home：首屏日历正常加载，日程组颜色、顶部入口、待办四象限、提醒、Agent 历史与输入区均可见；文档无横向溢出。
- 全局搜索：焦点自动进入主输入，类型与范围过滤布局清晰，背景层与弹层层级正确。
- 设置：基本设置仅显示基本分类；切换 AI 设置后只显示 Agent 配置、上下文优化和 Token 用量，复选框与保存按钮对齐正常。
- 文件管理：真实文件夹和文件列表正常横排，长文件名可读，操作按钮独立占位，使用量与视图切换保持可见。
- 分享组：四个真实分享组以两列卡片展示，查看日程、复制 ID、退出/删除动作均保留。
- 本轮浏览器检查只读取页面，没有创建、修改或删除用户数据，也没有保存设置。
