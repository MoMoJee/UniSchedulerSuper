## 5. P4-A 实际规划

P4-A 将基于本阶段基线实施：

1. 新增可信 `PlannerExecutionContext`、DTO、异常与结果 presenter。
2. 新增 `PlannerApplicationService`，先覆盖 normalized event/todo/reminder/group/share/search/conflict use cases。
3. V2 View 与 service 共享同一 use-case 层；不得让 service 调用 View，也不得构造 MockRequest。
4. 加入 rollout entrypoint 和 user/ref 权限验证；normalized 禁止 legacy fallback。
5. 以 service 直调和 HTTP adapter 的对照测试验证结果、版本、ChangeSet 一致。
6. P4-A 尚不切换现有 Agent tools；切流属于 P4-C，以便问题可定位、阶段可回退。
