# 样式规范

> 现行版本：2026-07-13。本文约束新增与修改样式；保留历史样式不应被误解为新的颜色或组件 API。

> 描述 UniSchedulerSuper 前端 CSS 编写约定、主题变量体系和 Bootstrap 使用规范。

## 1. CSS 变量（主题变量）体系

所有颜色、阴影、背景色必须使用 CSS 自定义属性（变量），**禁止硬编码颜色值**。

### 1.1 变量定义位置

定义在 `core/static/css/home-styles.css` 的 `:root` 和 `[data-theme="xxx"]` 选择器内。

### 1.2 变量一览

#### 背景色

| 变量 | 亮色模式值 | 用途 |
|------|-----------|------|
| `--bg-primary` | `#ffffff` | 主背景（卡片、Modal） |
| `--bg-secondary` | `#f8f9fa` | 次级背景（侧边栏、面板） |

| `--bg-body` | `#f5f5f5` | 页面 body 背景 |
| `--bg-hover` | `#f8fafc` | 鼠标悬浮背景 |

## 2. 主题系统

### 2.1 可用主题

`ThemeManager` 管理 13 个主题，通过 `data-theme` attribute 切换：

```
light（默认）、dark、auto（跟随系统）
china-red、warm-pastel、cool-pastel、macaron、dopamine
forest、sunset、ocean、sakura、cyberpunk
```

### 2.2 主题应用方式

`ThemeManager.applyTheme(theme)` 设置 `<html>` 的 `data-theme` attribute：

```javascript
document.documentElement.setAttribute('data-theme', effectiveTheme);
```

CSS 每个主题在 `home-styles.css` 中对应一个 `[data-theme="xxx"]` 规则块，覆盖 `:root` 变量。

### 2.3 新增主题

在 `home-styles.css` 末尾添加 `[data-theme="my-theme"]` 规则，覆盖所有 `:root` 变量；  
然后在 `ThemeManager.themes` 数组中添加主题名称。

## 3. 编写新 CSS 的规则

### 3.1 使用变量，带 fallback

```css
/* ✅ 正确：使用变量并提供 fallback */
.my-card {
    background-color: var(--card-bg, #ffffff);
    color: var(--text-primary, #2d3748);
    border: 1px solid var(--border-color, #e2e8f0);
}

/* ❌ 错误：硬编码颜色 */
.my-card {
    background-color: #ffffff;
    color: #333;
}
```

### 3.2 Bootstrap 优先

优先使用 Bootstrap 5 工具类，只在必要时编写自定义 CSS：

```html
<!-- ✅ 优先用 Bootstrap 工具类 -->
<div class="d-flex align-items-center gap-2 p-3 mb-2">

<!-- ❌ 避免：自定义 CSS 实现 flex 布局 -->
<div class="my-flex-row">
```

### 3.3 响应式

使用 Bootstrap 断点工具类处理响应式，避免手写 `@media` 查询：

```html
<div class="col-12 col-md-6 col-lg-4">
```

必须自定义 `@media` 时，优先使用 Bootstrap 断点变量：

```css
@media (max-width: 768px) {
    .sidebar { display: none; }
}
```

## 4. CSS 文件结构

```
core/static/css/
├── home-styles.css      # 全局样式、CSS 变量、主题定义（所有页面共用）
├── agent-chat.css       # Agent 对话面板样式
├── share-groups.css     # 分享组弹窗样式
├── quick-action.css     # 快速操作按钮样式
└── ...
```

**规则**：
- 每个功能模块的样式放在独立 CSS 文件中，不与 `home-styles.css` 混写
- 新增 CSS 文件后，在对应模板的 `<head>` 中添加 `<link>` 并附版本号

### 4.1 Token 百分比堆积图

- Agent 用量可视化统一使用三段 token：`input_cache_miss_tokens`、`input_cache_hit_tokens`、`output_tokens`。
- 百分比堆积图样式放在 `agent-chat.css`，上下文详情与 AI 设置页可复用同一组 token 颜色语义。
- 颜色必须使用主题变量：cache miss 用 `--warning`，cache hit 用 `--success`，output 用 `--info`，并提供合理 fallback。
- 图表只展示聚合 token、费用、调用点和 source，不直接展示 `AgentUsageRecord.diagnostics`。

## 5. 滚动条样式

`home-styles.css` 统一定义滚动条样式，通过 `--scrollbar-*` 变量跟随主题切换：

```css
::-webkit-scrollbar { width: 6px; height: 6px; }
::-webkit-scrollbar-track { background: var(--scrollbar-track); }
::-webkit-scrollbar-thumb { background: var(--scrollbar-thumb); border-radius: 3px; }
```

各组件内滚动容器不需要单独定义滚动条样式。
