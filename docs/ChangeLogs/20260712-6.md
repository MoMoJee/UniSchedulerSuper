# 2026-07-12 P4 Agent、MCP、附件与会话回滚升级

- 建立统一 `PlannerExecutionContext` 与 `PlannerApplicationService`，Web、Agent、Quick Action、MCP、附件共享同一 normalized use-case 层。
- Agent/Quick Action/MCP 的 normalized 工具不再自行操作 ORM/RRule 或 legacy Planner JSON；cache 保存完整 occurrence ref 并隔离 user/session。
- 新增服务端 rollback window、压缩 aggregate before-snapshot、hash 冲突检测与按消息倒序恢复；切走会话立即使旧窗口和 snapshot 失效。
- 前端移除 localStorage 的回滚授权职责；history 返回 floor/window/can_rollback，旧窗口及升级前回滚请求统一返回 410。
- internal attachment 对 normalized cohort 走 application query，并固化源删除后仍可读的 immutable snapshot。
- Quick Action 注入真实 per-tool ID，补齐同步 timeout/取消；MCP stdio/HTTP 隔离认证用户、request 与 transport session。
- 清理 42,259 个旧 Version、2,075 条旧事务及 17,213 个 orphan Revision；P3 ChangeSet 仅保留轻量 affected-ref 审计。
- 当前 SQLite 经 `VACUUM INTO` 从约 2.42 GB 缩小到 32.08 MB，业务 checksum、32 用户 strict/parity、integrity/foreign key 均零差异。
- 修复 Agent 回滚窗口只依赖新版前端 `rotate` 请求的问题：WebSocket 建连与 history 读取均会在服务端按当前消息末尾安全补建窗口，刷新复用原窗口，切换会话关闭旧窗口；即使未发生 Planner 工具调用，当前窗口内的普通对话消息也可回滚。
- 严格存储验收改为精确检查 Planner UserData key 与旧 `Before:` Agent 修订；非 Planner 的 `user_interface_settings` 正常历史不再被误报，同时任何新 legacy Planner JSON 版本仍会阻断验收。
- 最终自动化 `core.tests + agent_service.tests` 为 124/124 通过；Django check、migration drift、JS 语法、生产库 strict storage 和 diff 检查通过。
