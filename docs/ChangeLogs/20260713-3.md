# 2026-07-13 生产发布后缓存与 VariFlight 兼容修复

## 修复

- 首页与 `/api/v2/planner/bootstrap/` 复用同一份 cohort 决策，并将 browser entrypoints 安全内嵌到 HTML，消除 Planner 客户端异步初始化空窗。
- Planner V2 客户端在 entrypoint 未知或 bootstrap 失败时改为 fail-closed，不再把未知状态当作 legacy，从而避免 P6 封存后误调用 legacy Planner 写接口。
- 首页增加 `no-store`，避免浏览器通过页面缓存或 BFCache 长时间恢复旧的静态资源引用。
- 首页 CSS/JS 从人工维护的日期版本号切换为文件内容 SHA256 自动版本，可继续安全使用 Nginx `immutable` 长缓存。
- VariFlight MCP 结果解析兼容新版 `langchain-mcp-adapters` 返回的 `list[text content block]`，并以 `ast.literal_eval` 替代 `eval` 解析带前缀的 Python literal 响应。

## 验证

- 首页内嵌 entrypoints 与 bootstrap API 完全一致。
- normalized 首页不创建 legacy calendar 业务行。
- 静态资源内容变化会自动改变 URL 版本。
- Planner bootstrap 未完成/失败时不会回退 legacy。
- VariFlight content-block list、直接 data list 和完整公开工具格式化均通过回归测试。
