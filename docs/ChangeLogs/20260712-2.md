# 2026-07-12 Planner P3-B：versioned event command 与窗口搜索

## 新增能力

- 新增 `PlannerCommandService`，作为 normalized event v2 写入的唯一入口。
- 新增 `POST /api/v2/events/`，创建单次 event 或 recurrence master；RRULE、RDATE、EXDATE 分别校验和存储，正常 occurrence 不落 `CalendarEvent` 行。
- 新增 `PATCH /api/v2/events/{event_id}/` 与 `DELETE /api/v2/events/{event_id}/`：
  - 单次 event 使用 `single` 或 `all` scope 更新/软删除实体；
  - recurrence 的 `all` 更新 master/series 或软删除二者；
  - recurrence 的 `single` 必须带完整 `occurrence_ref`，修改写稀疏 `modified` override，删除写稀疏 `cancelled` override。
- 新增 `GET /api/v2/search/?q=&from=&to=&page=&page_size=`，在有限时间窗中展开并返回 event occurrence 与稳定 ref；当前只支持 `types=event`。
- 修复 `PlannerTimeCodec` 对 ISO DATE（如 `2026-03-01`）误判为午夜 datetime 的问题；全天 event 的 DATE 语义现可由 v2 创建接口完整保留。

## 一致性与安全边界

- 所有 v2 读写仍只允许 P2 verified 且无未解决 migration issue 的用户；未达标者统一得到 `409 planner_normalized_not_verified`。
- PATCH/DELETE 强制 `expected_version`（body 或 `If-Match`），陈旧写入返回 `409 version_conflict`。
- 每个成功 command 都在一个事务中创建 `PlannerChangeSet`，并递增默认 event collection 的 `CalendarCollectionVersion`、写入 `CalendarChange`。
- 未实现未来 override 归属策略前，`this_and_future` 统一返回 `409 recurrence_split_requires_override_policy`；不会悄悄截断或分裂 series。
- v2 不回退 legacy、不双写 legacy JSON、不创建重复实例；旧 Web URL 和 FullCalendar 没有切流。
