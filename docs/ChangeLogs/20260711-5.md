# 2026-07-11 Planner P1-B：legacy 调用入口收敛完成

## 变更摘要

- 新增 `PlannerUserDataCompat` 与 `PlannerUserDataRecord`，为复杂旧入口提供窄兼容层；Planner key 的 `get_or_initialize`、`objects.get/get_or_create`、`get_value/set_value` 保持旧调用形状，但不再触发 `DATA_SCHEMA` 重建。
- events、reminders、共享日程组、订阅日历、Agent parser/tool 和 CalDAV event 入口改为使用该兼容层；Calendar subscription、CalDAV 基础读取直接使用 `LegacyPlannerRepository` 的只读方法。
- `EventGroupService` 读取日程组时不再因读取创建空 key；创建时在事务内锁定并更新 legacy group 列表。
- Agent 事务回滚快照改由 `LegacyPlannerRepository.get_rows_for_revision()` 提供，持续覆盖 events、todos、reminders、规则段和既有 `outport_calendar_data` 导出数据。
- 调用面报告新增 `out_of_planner_scope_count` 与 `planner_bypass_count`，明确区分 Agent 配置/用量、用户偏好与 Planner 实体访问。

## 数据与兼容性

- 没有迁移、删除或重写任何生产 Planner 业务数据；legacy JSON 仍是唯一事实源。
- URL、请求字段及成功/错误响应格式保持兼容。
- 兼容写入按原 JSON 序列化，保留未识别字段；重复 source key 或非法 JSON 会显式失败，不会用默认值覆盖。
- normalized 表在本阶段仍不承接任何线上读写流量。
