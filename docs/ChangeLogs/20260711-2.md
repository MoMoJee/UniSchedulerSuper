# 2026-07-11 Planner P1-A 查询基础与 P1-B TodoService 首批收敛

## 变更摘要

- 新增 `core/planner/repository.py`，提供 normalized ORM 的 event definition 与 occurrence 查询投影。
- repository 使用 EventRecurrenceSeries、RDATE、EXDATE、sparse override 构造 `RecurrenceDefinition`，统一委托纯 `RecurrenceExpander`；查询不会创建任何普通 occurrence 行。
- 为 `PlannerVersionedModel` 新增 `require_expected_version()`、`bump_version()`、`soft_delete()`，为后续 command 的 409 乐观锁、软删除和版本递增建立统一基础。
- 新增 `report_planner_direct_userdata_access` 管理命令，扫描 `core`、`agent_service`、`caldav_service` 中的 `UserData.objects`、`get_or_initialize()`、`set_value()`；测试目录、迁移目录和明确白名单不计入待收敛项。
- 当前调用面报告为 164 处非白名单直接访问，作为 P1-B 的基线。
- 扩展 `LegacyPlannerRepository`：支持事务内 `select_for_update()` 读取/初始化列表 key，以及不经 `DATA_SCHEMA` 重建的 `replace_list()` 写入。
- `TodoService` 已切换到 legacy repository：读取不再隐式初始化；创建、更新、删除在 `transaction.atomic()` 与 `reversion` 内集中读写 legacy JSON；未知字段保持不变。
- 调用面报告确认 `core/services/todo_service.py` 的直接 UserData 访问为 0。

## 数据影响

- 未导入、未重写任何正常 Planner 业务数据。
- TodoService 的 legacy JSON shape 与 UUID、字段名、reversion comment 保持兼容；仅改变内部访问路径。
- normalized 表仍未接入任何现有 API、Agent、MCP、Feed 或 CalDAV 请求。
