# 2026-07-13 P4-G Agent 附件与 Reminder 完整链路修复

- Agent 内部 Event/Todo/Reminder 附件统一使用 master entity ID，并携带 occurrence_ref 固化具体实例的有效时间与 override 快照。
- 附件创建、格式化和 WebSocket 消费均执行 user/session/pending/数量一致性校验；任何失败中止发送，不再生成“UI 有磁贴、模型无附件”的假成功。
- 回滚目标消息的附件改为原子退回 pending，后续消息附件才软删除；前端只使用 rollback response 的 authoritative DTO 恢复发送框，文件/图片可用同一附件 ID 重新发送。
- 修正附件绑定的 LangGraph message_index，使其与实际 HumanMessage 索引一致。
- Reminder v2 支持单次转重复、重复转单次、RRULE 替换，以及 single/all/this_and_future 范围编辑和删除；all 允许改时刻但拒绝改变系列起始日期。
- Reminder 侧栏改为按所选有界窗口展开 occurrence，并保留 occurrence_ref/source_version；日历、详情、编辑表单与侧栏使用同一引用和统一刷新入口。
- 修复已软删除 recurrence series 仍让 Reminder 被排除在单次查询之外的问题。
- 修复 FullCalendar 同窗口并发请求用空数组覆盖有效结果的问题；Reminder 筛选不再隐式重复刷新日历。
- 本地验收服务改用 ASGI/Daphne，恢复 `/ws/agent/` WebSocket；普通 HTTP runserver 不作为 Agent UI 验收入口。
- 更新前端静态资源版本并重新 collectstatic；新增 Reminder scope、附件首发/回滚重发/跨会话拒绝和 rollback endpoint 集成测试。
