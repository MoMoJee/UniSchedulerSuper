# 2026-07-11 Planner P1-B：Todo Web API 入口收敛

## 变更摘要

- `GET /api/todos/` 改为通过 `TodoService.get_todos()` 读取；缺失 `todos` key 时返回空列表，不再因读取隐式创建源行。
- `POST /api/todos/create/`、`POST /api/todos/update/`、`POST /api/todos/delete/` 改为通过 TodoService 和 LegacyPlannerRepository 执行。
- Todo Web API 的 URL、认证、请求字段、成功/404 响应 JSON 保持兼容。
- `POST /api/todos/convert/` 改为在单一 `transaction.atomic()` 与 reversion revision 中锁定 `todos`、`events` 两个 legacy 列表，创建 event 并更新原 todo 的 converted 状态与关联 ID。
- 所有 Todo Web 写入不再调用 `DATA_SCHEMA` 重建，保留 legacy JSON 的未知字段。

## 数据影响

- 未迁移、未重写任何已有业务数据。
- 仅当用户调用 Todo API 时，内部读写路径变为 LegacyPlannerRepository；返回格式保持不变。

## 验证

- 新增 Todo HTTP 回归测试，覆盖空列表读取、HTTP CRUD、未知字段保留和 todo→event 原子转换。
- `python manage.py test core.tests.test_todo_views_legacy_adapter`：3 项通过。
