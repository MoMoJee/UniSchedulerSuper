# 2026-07-15 React 工作台 R0–R5 优化升级

## 升级范围

本次完成 `React旧版工作台复刻与信息架构收敛修复方案.md` 的全部 R0–R5。登录后的产品信息架构已收敛为一个 `/home/` 工作台；Todo、Reminder、Calendar、Agent 常驻，搜索、设置、分享组和文件通过 overlay/surface 打开，不再存在另一套全页业务壳。

## 分阶段改动

### R0：容器与状态

- 新增可访问的中央模态框，支持遮罩、Escape、关闭回焦、固定标题/底栏和内部滚动。
- 新增文件用全屏 surface 和 URL 可恢复的 workbench surface store。
- 日程、提醒、日程组、详情、Agent 工具选择及文件移动统一迁出通用右侧 Sheet；Sheet 只保留附件/文件预览等侧栏语义。

### R1：Todo 与 Reminder 左栏

- Todo 支持列表/四象限视图、状态与重要性筛选、持久化视图选择、完成/恢复、编辑、删除、转日程。
- Reminder 支持今天/7 天/30 天范围、重复筛选、中央系列编辑和基于 `occurrence_ref` 的完成、忽略、延后。
- 所有写入继续只走 `/api/v2/` 且携带版本；成功后定向失效 Planner query，不刷新/卸载 Agent。

### R2：Calendar 和分享投影

- 常驻的组复选框改为筛选 popover；Calendar 主体高度不再因筛选项数量变化。
- 分享组改为紧凑横向 tab。
- 修复共享日程与个人 occurrence 相加的问题；分享 scope 现在只显示 `/api/v2/share-groups/<id>/occurrences/` 返回值，不再回落个人事件或提醒，也不再发出个人 occurrence/definition/reminder 查询。

### R3：灵动岛与搜索

- 顶部统一为搜索、设置和更多入口；更多菜单固定包含文件、设置、帮助和带确认的退出。
- 搜索增加类型、个人日程组、分享组和时间窗口；结果打开同一 Home 的对应日期/实体，而不是跳到重复 workspace。

### R4：分享组和文件

- 分享组中央管理框保留 owner/member 权限动作，增加复制组 ID，并可直接切换日历 scope。
- 文件使用覆盖式完整工作区，桌面为目录树+内容，窄屏改为单列；保留上传、URL 上传、文件夹、预览、Markdown、下载、重命名、移动和删除。

### R5：设置和路由

- 设置改为基本设置、日程偏好、显示偏好、提醒设置、AI 设置、我的六 tab。
- 保留 13 个主题、系统主题和金色装饰开关；即时预览不等于保存，取消会恢复打开前主题。
- `/home/todos|share|search|files|settings/` 兼容链接统一重定向 `/home/` 或对应 `surface`。
- UI store 升到 v3，将旧实验面板比例一次性迁移为 20/50/30；之后用户调整仍会持久化。
