# 2026-07-12 P3-D 重复日程作用域与分享组 UI 修复

## 用户验收发现的问题

- V2 occurrence adapter 没有向旧 UI 暴露 `rrule`、`series_id` 和 `is_recurring`，详情页不显示规则，删除流程会把重复实例误判为普通单次事件。
- 弹窗批量编辑/删除仍固定调用 `/api/events/bulk-edit/`；normalized cohort 的后端硬阻断正确拦截了该 legacy JSON 写入。
- V2 分享事件取回后又经过当前用户的个人日程组筛选；他人事件的 `groupID` 不在当前用户集合内，因而被全部过滤。
- 前一轮浏览器 smoke 只覆盖普通 CRUD、拖拽和跨窗口读取，没有覆盖“点击详情→选择范围→编辑/删除”的完整交互矩阵，因此没有及时发现上述调用残留。

## 修复内容

- `planner-v2-client.js` 补齐 recurrence、主事件时间、DDL、位置、状态、分享关系等兼容字段。
- `event-manager.js` 的弹窗 single/all/future/from-time 编辑和删除在 normalized cohort 全部改走 V2 command；legacy cohort 仍保留旧路径。
- 新增可独立测试的 `planner-v2-scope.js`：
  - single 可改日期和时间，但不隐式改规则或系列分享关系；
  - all/this-and-future 可改时间点和时长，不允许改日期；
  - all 将所点 occurrence 的时间差换算回 master；
  - 只有用户明确进入“更改重复规则”模式才提交 recurrence，普通 future 编辑按原规则剩余次数切分。
- 后端允许单次 Event 显式转为 recurrence；补齐有限↔无限和 future 停止重复。
- 系列整体平移时同步迁移 Override、EXDATE、RDATE 和 split review 的 `RECURRENCE-ID`；有效时间随之平移，避免旧键形成额外 occurrence。
- future 时间切分时真正按 ordinal 搬迁既有 override，并同步未来稀疏日期关系。
- Event create/all/future command 支持 `share_group_ids` 的权限校验和关系替换；definition 回传当前分享关系，普通编辑不再意外取消分享。
- 分享组 V2 查询按 `event_ids` 限缩 recurrence 展开，并回传成员、owner color、规则和 master 时间；V2 分享视图不再套用个人日程组筛选。
- 分享组日期窗口切换会重新读取当前群组。旧实现把点击标签时的窗口结果注册成静态 FullCalendar source，翻周/月后不会请求新窗口；现由 `datesSet` 在仍处于同一群组时安全重载，快速切换标签不会被迟到请求覆盖。
