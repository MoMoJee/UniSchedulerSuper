# 2026-05-24 上下文构建详情父子调用树

## 变更摘要

- 后端 `last_llm_request_snapshot` 改为父子结构：一次用户请求作为父级调用，每次 LLM 调用写入 `child_snapshots`。
- 父级快照继续展示最终完整上下文，父级 `token_stats` 改为所有子调用 token 求和。
- 子调用保留真实该轮上下文和真实该轮 token，用于逐轮排查工具循环。
- 前端“请求历史”改为父子树，父级可展开查看每一轮 LLM 子调用。
- UI 模式和 JSON 模式共用选中快照解析逻辑，点击子调用时两种模式都展示子快照。
- 更新 `agent-chat.css` 静态版本号为 `20260524-001`，`agent-chat.js` 静态版本号最终为 `20260524-003`。
- 修复父子快照 token 写入分支中 `token_stats` 局部变量未定义导致的保存失败。
- 修复项目 logger 对 `exc_info=True` 的处理，避免异常日志触发 logging 二次错误。
- 修复上下文构建详情仅依赖 localStorage 导致刷新或切会话后空白的问题，打开弹窗时从后端 `llm_request` 兜底加载最新快照并自动选中。
- 修复上下文可视化接口中 `AgentTransaction` 旧字段查询导致的工具信息 warning。
- 修复大上下文快照写入 localStorage 时触发 `QuotaExceededError` 的问题，完整快照改为当前页面内存保存，localStorage 仅写入瘦身缓存。
- 更新前端/后端规范文档，补充大对象缓存、后端兜底恢复、父子快照结构和异常日志 traceback 规则。

## 经验规则

- 上下文可视化的左侧“请求历史”应以用户消息为父级，而不是以单次 LLM 调用为父级。
- 父级快照可以复用最后一轮完整上下文，但 token 必须来自全部子调用聚合。
- JSON 模式不能只展示父级最终快照；选择子节点时必须展示对应子调用快照。
- 分支内更新后的状态日志应读取最终容器字段，避免引用只在某个分支赋值的局部变量。
- 上下文详情不能只依赖 WebSocket 写入的本地缓存；打开弹窗时必须能从后端 `last_llm_request_snapshot` 恢复。
- LLM 请求快照可能包含工具定义、完整消息和图片占位，不能把完整快照长期写入 localStorage；前端应使用内存显示完整数据，只持久化瘦身摘要。
- WebSocket 推送不是上下文详情的唯一数据源；刷新、切会话、本地缓存丢失时必须通过后端只读接口恢复。
- 后端异常需要 traceback 时使用 `logger.error(..., exc_info=True)`，由项目 logger 统一处理。
