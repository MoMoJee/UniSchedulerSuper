# 2026-07-11 Planner P1-B：ReminderService 与 EventService 收敛

## 变更摘要

- `ReminderService` 的 reminders 主列表读写已切换到 `LegacyPlannerRepository`。
- Reminder 的普通创建、更新、删除在 `transaction.atomic()` 与 `reversion` 内执行，避免 `DATA_SCHEMA` 重建删除未知字段。
- 重复提醒仍使用现有 `IntegratedReminderManager` 与 `rrule_series_storage` 兼容路径；本轮未改变重复提醒实例生成、规则段和例外语义。
- `EventService` 的 events 主列表读写已切换到 `LegacyPlannerRepository`。
- Event 的普通创建、更新、删除和 bulk-edit 前的目标查找均通过 legacy repository；旧 `EventsRRuleManager` 与 `events_rrule_series` 规则段存储暂保持兼容。
- EventService 保留一处 `user_preference` 读取，用于既有自动 DDL 配置；该 key 不属于本期 Planner 实体迁移范围。
- 调用面报告更新：排除测试代码后由 164 降为 145；TodoService、ReminderService 为 0，EventService 仅剩非 Planner 的 `user_preference` 读取。

- 现有 Web views、分享/订阅、Agent 附件、CalDAV 仍未收敛，继续使用 legacy 路径。
