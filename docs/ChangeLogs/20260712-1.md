# 2026-07-12 Planner P3-A：v2 只读 definitions / occurrences API

## 新增能力

- 新增 `GET /api/v2/events/definitions/?from=&to=`，输出 normalized event definition 与 recurrence 元数据。
- 新增 `GET /api/v2/events/occurrences/?from=&to=`，按请求窗口纯展开 occurrence，并返回稳定复合 `occurrence_ref`；客户端展示 ID 只是列表 key，不可作为持久化实体 ID 使用。
- 新增 v2 presentation serializer，统一输出时区时间、全天事件、分组、状态、版本和 override 标记。

## 安全边界

- v2 只读接口仅允许 P2 已 verified 且无未解决 migration issue 的用户访问；否则返回 `409 planner_normalized_not_verified`。
- v2 不回退 legacy、不写数据库、不补实例；旧 `/get_calendar/events/` 和当前前端请求路径没有改动。
- 当前未为任何用户登记 v2/shadow 入口，因此没有生产流量切换。

## 验证

- 新增 API 回归测试，覆盖未验证用户拒绝、recurrence occurrence ref、definition metadata、非法时间窗和读取零写入。
- `manage.py test core.tests.test_planner_v2_api core.tests.test_planner_repository`：通过。
