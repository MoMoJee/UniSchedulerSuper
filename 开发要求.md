# 开发要求

## 需求背景
用户在使用 /home 页面的 Agent 栏时，可以添加附件、写入文本要求等。但刷新页面后，原先输入的内容会丢失。需要添加持久化保存逻辑。

## 功能需求
1. **文本内容保存**
   - 用户在 Agent 栏输入的文本内容，在输入后自动保存到 localStorage
   - 刷新页面后，自动恢复之前保存的内容

2. **附件保存**
   - 用户添加的附件信息（文件名等）需要保存到 localStorage
   - 刷新页面后，显示之前添加的附件列表
   - 注意：附件文件本体不需要持久化，只需要保存附件的元信息（文件名等）

3. **清除功能**
   - 提供"清除"按钮，一键清空已保存的内容
   - 清除后同时清空 localStorage 和页面显示

## 技术要求
- 使用 localStorage 实现前端持久化
- 兼容现有的 Agent 栏 UI
- 不影响现有的 Agent 对话功能

## 约束条件
- 不要修改后端代码（Django）
- 不要破坏现有的 Agent 对话流程
- 保持前端代码风格一致
